01 — The problem

You're driving a sports car
to the mailbox.

Using AI as a chatbot barely scratches the surface. What's changed isn't that AI got smarter — it's that we've figured out how to work with it.

The breakthrough isn't better models. It's better workflows. Break big problems into small focused pieces. Create plans and specs. Orchestrate multiple AI agents the way a project manager coordinates a team.

This approach works for code — but it works equally well for any structured project.

02 — The methodology

One pattern. Any project.

The spec-driven workflow isn't a software technique. It's a project management methodology that happens to work incredibly well with AI. If you can describe a goal and break it into parts, you can use it.

Step Personal Website Apartment Renovation
Vision A site that showcases my work and lets me blog A comfortable, modern apartment that feels like home
Break down Homepage, About, Blog, Projects, Contact Living room, Kitchen, Bedroom, Bathroom, Hallway
Research AI researches design trends, CMS options AI researches materials, prices, regulations
Plan Feature list, tech stack, phased rollout Room-by-room budget, timeline, dependencies
Execute "Build hero section," "connect CMS," "deploy" "Get 3 quotes," "order tiles," "schedule electrician"
Verify AI self-checks code, runs tests, you review Review quotes vs budget, check samples, adjust
The point is not that AI will tile your bathroom. The point is that the hard part of any project — the structuring, research, planning, tracking — is exactly what AI is good at when you give it the right workflow.
03 — The evolution

How we got here.

Each stage solves a limitation of the previous one. The final insight: the breakthrough isn't better models — it's better workflows.

Chat / LLM
Instant knowledge, writing, brainstorming
Can only talk. Can't do anything.
Agent
Multi-step reasoning. Plans and iterates.
Sandboxed. No access to your tools.
MCP & Tools
Reads email, searches files, calls APIs
Cloud only. Can't touch your machine.
Claude Code
Read, write, run, test code — your terminal
Context window limits.
Spec-Driven
Reliable execution of complex projects
Manual orchestration still required.
GSD →
You become the conductor. Agents work in parallel, self-verify.
The frontier. Where we are today.
